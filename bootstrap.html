Balises HTML de modification du texte

Taille, police et la couleur du texte

Dans les premières version du HTYML, il existait la balise <font> qui permettait de changer la taille, la police et la couleur d'une section de texte. Mais cette balise n'est plus dans les normes HTML, désormais pour ce genre de manipulation, on créera des classes CSS, qui seront utilisées ponctuellement pour décorer au besoin des section de texte.

On utilise le plus souvent la balise <span>, qui recevra la classe pour changer l'apparence du texte.

<span> ne sert qu'à faire ressortir visuellement du texte, pour avoir un impact sur le référencement, il existe d'autre balises.

Le gras et l'italique

Pour mettre en gras ou en italique, il existe pour chacun des cas deux balises. Les balises qui ne servent qu'à attirer l'attention du lecteur sont <b> pour le gras et <i> pour l'italique.
Pour attirer l'attention du lecteur ET signifier une importance sémantique qui impacte le référencement, on utilisera <strong> pour le gras et l'importance sémantique sur la force du terme et <em> pour l'italique et ajouter de l'emphase à la sémantique du terme.

D'autres mise en forme de texte

A la base, le HTML servait principalement pour l'impression directe de son contenu, c'est pour cela qu'il est toujours possible de souligner, de biffer ou de surligner au fluo du texte.
On ne souligne jamais en HTML, car les utilisateurs pourraient pense qu'il s'agit d'un hyperlien, et ils pourraient pense qu'il s'agit d'un lien cassé.

Les polices de caractères

La police par défaut des navigateurs est faîte pour de l'impression directe, avec empâtement (type Times New Roman), en revanche, sur des écran rétro-éclairés il sera plus confortable d'utiliser une police sans empâtement (type arial ou helvetica)

Les tailles de polices et le problème du em

Le em représente la taille par défaut d'une police sur la page dans un paragraphe.
Quand on utilise par exemple h1, la taille généralement appliquée par le navigateur est de 2 fois le cadratin

Mais le em pose problème : il se calcule par rapport à la taille de la police du parent le plus proche.

Dimensions web et CSS

Par défaut dans le css user agent (css du navigateur), quand on donne une taille (hauteur ou largeur) à un élément, la taille est calculée par rapport au contenu de l'élément, c'est à dire qu'elle ne prend pas en compte la taille des marges intérieures (padding) ou de la largeur des bordures.

Les positions en CSS

La propriété position permet d'afficher à l'écran des éléments ailleurs que là où ils sont déclarés dans le fichier HTML.

les valeurs :

  • static : par défaut pour les éléments
  • Relative : l'élément est à sa position déclarée mais peut être décalé à l'aide des propriétés de type top, bottom, le décalage n'a pas d'impact sur les éléments suivants
  • absolute : l'élément n'a plus de "contenance", les éléments suivants passeront "au travers" ou "en-dessous". Les propriétés de type top, bottom... le placeront par rapport son ancètre le plus proche ayant la position relative.
  • fixed : l'élément n'a plus de "contenance", les éléments suivants passeront "au travers" ou "en-dessous". Il est positionné par rapport au viewport et ne bougera pas quel que soit le défilement de la page.
  • sticky : l'élément est tout d'abord placé selon son élément parent, mais lors du défilement vers le bas, bougera jusqu'à la disparition du viewport de l'élément parent, ensuite, il reste en haut du viewport avec le décalage indiqué par rapport à son élément parent.

Le CSS adaptatif

Le CSS dit adaptatif (ou plus communément appelé RWD - Responsive Web Design) permet d'adapter l'affichage d'un site sur selon l'appareil utilisé.

Donc, un site s'affichant correctement sur smartphone (en prenant en compte la lisibilité et l'accessibilité) doit pouvoir s'adapter sous une forme différente sur des écrans plus grands (tablette ou PC)

Pour ce faire, on va écrire dans le CSS des surcharges ou des nouvelles règles CSS quand la largeur de l'écran atteint ou dépasse une largeur définie, en utilisant les media queries

Les media-queries

Les media queries permettent de surcharger ou d'ajouter des comportements CSS par rapport, entre autres, à l'affichage du site sur l'appareil : largeur d'affichage (viewport), l'orientation (portrait ou paysage), etc.

Il est recommandé d'écrire le CSS d'un site en smartphone-first. La plupart des visiteurs de site naviguent à l'aide d'un smartphone. De plus le CSS de base sans media-queries est déstiné uniquement à l'affichage sur smartphone. Cela permet d'écrire quasiement tout le CSS d'un site, les surcharges ou ajouts de CSS dans les media-queries ne représentant qu'une partie infime du fichier CSS total.

Si on fait la mise en page en premier lieu pour un écran plus grand, les réécriture et surcharge de CSS seront beaucoup plus importante, ce qui génèrera un fichier CSS très gros et surtout très difficile à maintenir.

la propriété display

La propriété display permet de contrôler le mode d'affichage d'un élément sur la page et parfois même au sein d'un "flux" de texte, généralement dans un paragraphe.

Les valeurs possibles (liste non exhaustive) :

  • block : l'élément prend toute la largeur disponible et "repousse" les éléments (ou le flux dee données) au dessus (pour ce qui arrive avant) et en dessous (pour ce qui arrive après)
  • inline : l'élément prend la largeur de son contenur et est intégré ans le flux de texte à la suite
  • inline-block : completement semblable au inline, mais généralement utilisé sur des éléments ayant la proriété float d'activée, afin de les aligner côte à côte
  • None : l'élément n'est tout simplement pas affiché à l'écran
  • flex : permet d'utiliser les propriétés flexible CSS (principalement utilisées pour le CSS adaptatif) MAIS pour les élément À l'intérieur
  • grid : permet de configurer une disposition en grille, en configurant le nombre et l'espace pris par les colonne, la gestion automatique ou forcée des lignes, et l'espace a prévoir entre les éléments
